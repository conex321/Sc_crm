**Non-negotiable rules for this plan:**
1. All hex values come from UI-SPEC Sections 1–3 VERBATIM. No sampling, no improvising — the palette is already verified from Pipedrive's live token CSS.
2. `@theme` layering rule (RESEARCH Finding 1, verified against Tailwind docs): **literal values** go in plain `@theme` (Layer 1 base scales); **runtime-switchable values** go in plain CSS `:root` / `.dark` (Layer 2 semantics); **var-references** that must resolve at the element go in `@theme inline` (Layer 3 bridge). Putting a `var()` reference in a plain `@theme` block resolves it at `:root` and silently breaks `.dark` switching.
3. Keep every existing shadcn variable NAME (`--primary`, `--card`, `--sidebar-accent`, …) — all 21 `components/ui/*` primitives consume them. Only values change.
4. `components/ui/sidebar.tsx` and all other `components/ui/*` primitives: ZERO edits in this plan (component deltas are PLAN-02). Composition files (`components/layout/*`) are the edit surface.
5. Restyle without rewording: page headings/labels on e2e-checked routes must keep their text (`scripts/e2e-rayan.mts` asserts body regexes). New copy (quick-add items, search placeholder) comes from the UI-SPEC Copywriting Contract exactly.
6. Density locked: 14px root, `text-xs`/`text-sm`, `h-8` controls, `px-6 py-5` wrappers. Only the top bar grows (42→48px).

**Layer 1 — base scales, new plain `@theme` block (literals only):**
Transcribe UI-SPEC Section 1 tables verbatim as:
- Light scales: `--color-pd-{green|blue|red|yellow|purple|neutral}-{0..1000}` (shade 0 exists only for neutral: `#ffffff`). E.g. `--color-pd-green-600: #2d8647;`.
- Dark scales: `--color-pd-dark-{color}-{0..1000}` (neutral-0 dark: `#0e1017`). E.g. `--color-pd-dark-green-600: #3c824e;`.
This generates `bg-pd-green-600` etc. utilities AND emits the vars for aliasing. ~130 declarations — write them all; Phase 2 chips/stage colors consume these scales.

**Layer 2 — semantic values, replace the existing `:root` and `.dark` blocks:**

Each pair in the spec is written `light / dark` — put the first value in `:root`, the second in `.dark`. Label chips in `.dark`: bg = `--color-pd-dark-{color}-200`, fg = `--color-pd-dark-{color}-800` (e.g. green `#013417`/`#76a97c`). Elevations: same values in both themes.

- PARITY RULE: every `--pd-*` var declared in `:root` MUST also be declared in `.dark` in this same commit.

**Layer 3 — extend the EXISTING `@theme inline` block (var-references only):**
Keep all current mappings untouched; append bridges so utilities exist for the new semantics, e.g.:
`--color-pd-positive: var(--pd-positive); --color-pd-positive-strong: var(--pd-positive-strong); --color-pd-positive-bg: var(--pd-positive-bg); --color-pd-positive-bg-light: var(--pd-positive-bg-light);` … repeat for warning/negative/info, `--color-pd-link`, `--color-pd-link-hover`, `--color-pd-text-secondary`, `--color-pd-text-muted`, all twelve `--color-pd-label-*`, and shadow bridges `--shadow-pd-button: var(--pd-elevation-button); --shadow-pd-raised: …; --shadow-pd-raised-hover: …; --shadow-pd-floating: …; --shadow-pd-overlay: var(--pd-elevation-overlay);`.
Result: `bg-pd-warning-bg`, `text-pd-positive-strong`, `text-pd-link`, `shadow-pd-raised` all work and flip with `.dark`.

**Inter font (`app/layout.tsx` + bridge):**
- `import { Inter } from "next/font/google";` → `const inter = Inter({ subsets: ["latin"], variable: "--font-inter" });`
- Body: `className={cn(inter.variable, "font-sans antialiased")}` (or template-string equivalent).
- In `@theme inline`: `--font-sans: var(--font-inter), ui-sans-serif, system-ui, sans-serif;`
- Keep `font-feature-settings` in `@layer base`. Do not change the 14px root.
